## 常见问题应对

### Q1: 为什么选择人脸识别应用？

> "人脸识别是典型的计算密集型、延迟敏感应用。它包含多个可分离的计算模块(检测、嵌入、搜索)，非常适合研究计算划分问题。"

### Q2: 云端执行有什么优势和劣势？

> "优势：减少移动端能耗，可利用更强计算资源，支持更大规模数据库。劣势：依赖网络连接，增加网络延迟，存在隐私考虑。"

### Q3: 如何选择最优划分方案？

> "取决于：(1) 网络条件 - 低延迟高带宽适合更多卸载；(2) 设备能力 - 低端设备更应考虑卸载；(3) 延迟要求 - 实时应用可能倾向本地执行；(4) 数据库规模 - 大规模可考虑云端搜索。"

### Q4: 实验有什么局限性？

> "局限性：(1) 测试设备和网络环境有限；(2) 未考虑能耗测量；(3) 云端服务器性能可能与生产环境不同；(4) 未实现真正的动态划分。"

## 故障应对预案

### 场景1: 服务器无法连接

1. 检查网络连接
2. 重启服务器
3. 切换到 Mode 0 (全本地) 继续演示
4. 使用备份的性能数据说明

### 场景2: 应用崩溃

1. 重启应用
2. 清除应用数据重试
3. 使用备用设备
4. 使用录制的演示视频

### 场景3: 识别效果差

1. 确保光线充足
2. 调整人脸角度
3. 使用预先录制的视频演示
4. 强调实验重点是性能测量而非准确率

### 场景4: 时间不够

优先保证以下内容：
1. Mode 0 和 Mode 1 的对比演示
2. 性能数据展示
3. 结论总结
